## 二、可选扩展方向

下面按"投入成本"和"回报价值"两个维度列出可选方向。

### A. 内容扩展（低成本，高价值）

适合当前阶段，不需要额外 API 费用。

#### A1. 练习题答案库
- **做法**：在 `solutions/` 目录下为每个 `EXERCISES.md` 提供参考实现
- **收益**：让读者能独立学习，降低维护者答疑成本
- **成本**：时间投入中等
- **推荐度**：⭐⭐⭐⭐⭐

#### A2. 博客文章系列
- **做法**：在 wumingmp.me 写 3-5 篇密码学入门文章，每篇对应一个阶段
- **收益**：为网站引流，建立个人品牌
- **成本**：时间投入高，但无金钱成本
- **推荐度**：⭐⭐⭐⭐⭐

#### A3. 中文术语表
- **做法**：新增 `GLOSSARY.md`，整理中英文对照（如 nonce→临时数、plaintext→明文）
- **收益**：降低中文读者门槛
- **成本**：低
- **推荐度**：⭐⭐⭐⭐

#### A4. 可视化图解
- **做法**：为 AES、RSA、DH 等算法画流程图（可用 Excalidraw / draw.io）
- **收益**：大幅提升理解效率
- **成本**：中等
- **推荐度**：⭐⭐⭐⭐

### B. 工程化提升（中等成本，长期价值）

让项目更像一个专业开源项目。

#### B1. 文档站点迁移到 MkDocs
- **做法**：用 MkDocs / Material 替代手写的 `docs/index.html`
- **收益**：自动生成导航、搜索、多页面文档
- **成本**：中等，需要学习 MkDocs
- **推荐度**：⭐⭐⭐⭐

#### B2. 增加类型注解和代码质量工具
- **做法**：
  - 全代码加 `typing` 类型注解
  - 引入 `mypy` 静态检查
  - 引入 `ruff` / `black` 格式化和 lint
  - CI 中跑这些检查
- **收益**：代码更专业，适合教学示范
- **成本**：中等
- **推荐度**：⭐⭐⭐⭐

#### B3. 发布到 PyPI
- **做法**：把 `crypto-lab` 封装成可 pip 安装的包
- **收益**：方便他人使用，提升项目影响力
- **成本**：中高，需要学习打包发布
- **推荐度**：⭐⭐⭐

### C. 互动与社区（高成本，潜在高回报）

#### C1. 在线 CTF 挑战
- **做法**：用 GitHub Issues 或静态页面设计闯关题
- **收益**：极大提升参与度和传播性
- **成本**：高，需要设计题目和自动判题
- **推荐度**：⭐⭐⭐⭐

#### C2. 视频教程
- **做法**：录制每个阶段的讲解视频，发到 B 站 / YouTube
- **收益**：触达更广受众
- **成本**：很高（时间 + 设备 + 剪辑）
- **推荐度**：⭐⭐⭐

### D. 高级技术方向（需要较多 AI / 学习时间）

#### D1. 常见攻击演示
- **做法**：
  - ECB 模式图片加密演示
  - Padding Oracle Attack 概念验证
  - RSA 低指数攻击
  - 彩虹表攻击
- **收益**：理解"什么是不安全的"
- **成本**：高，需要确保示例不会误导初学者用于非法用途
- **推荐度**：⭐⭐⭐

#### D2. 真实协议深度解析
- **做法**：TLS 1.3 握手流程、Signal Protocol、WireGuard 基础
- **收益**：打通理论与实际应用的最后一公里
- **成本**：高
- **推荐度**：⭐⭐⭐

## 三、关于 API 费用与付费策略的建议

### 当前情况

- API 额度快用完
- 7 月 6 日有新的订阅计划（subscription plan）
- 正在纠结：现在付费 vs 等到 7.6

### 建议：先暂停付费，等到 7 月 6 日

理由：

1. **项目核心已经完成**
   - 剩下的工作（练习题答案、博客文章、文档优化）大多不需要大量 AI 交互
   - 你自己完全可以独立完成 A1、A2、A3 这些方向

2. **新订阅计划可能更划算**
   - 距离 7 月 6 日只有不到一周
   - 提前付费可能错过新套餐的优惠或额度重置
   - 除非现在有紧急任务，否则等待是理性选择

3. **可以用这段时间做"零 API 成本"的事**
   - 写练习题答案
   - 写博客文章
   - 整理术语表
   - 画流程图
   - 读密码学书籍/视频（Christof Paar 的课程）

### 如果必须现在付费的场景

只有在以下情况下建议现在付费：

- 7.6 之前必须交付某个紧急项目
- 当前剩余额度连基本对话都支撑不了
- 新 plan 的价格/额度已经明确不如当前方案

否则，**等 7.6**。

## 五、给这个项目的定位建议

Crypto Lab 现在的最佳定位是：

> **"一个适合中文开发者自学的密码学动手实验室"**

不要把它做成：
- 全面的密码学教材（投入太大）
- 生产级加密库（责任太大）
- CTF 平台（需要大量运维）

保持"小而精、可运行、有练习"的特色，是它最大的竞争力。
